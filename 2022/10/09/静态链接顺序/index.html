静态链接的链接顺序

pid=68656805

本文翻译自[1]

规则

  1. 调用链接器时传入的 文件/库 的顺序就是链接顺序，链接器按顺序处理这些 文件/库
  2. 链接器会维护当前已经遇到的符号，包括已定义的符号，与未定义的符号
  3. 当遇到 object file 时
    • 对于 object file export 的符号，检查其是否已经出现过，若否，则加入已定义符号集合
    • 对于 object file import 的符号，如果其不存在于 已定义符号集合，那么加入 未定义符号集合
  4. 当遇到 库 时，遍历所有其包含的 object 文件，对每个文件，首先会获得其 export 的符号。如果 当前 object 文件的 export 符号集合 交 当前维护的 未定义符号集合 不为空，那么将当前 object 文件像步骤三一样处理（收集其未定义符号）。
  5. 如果在库中任何 object file 找到了未定义符号的定义，那么由于这个 object file 也会走第三部，可能会引入新的 未定义符号，且这些未定义符号可能已经在库内其他文件定义，因此在这种情况下会对这个库重新走一遍步骤四
  6. 对于库中那些对于填补未定义符号没有帮助的 object file，不会被链接到最终的 executable。这也是为什么 libc 中每个函数都表示为一个 object file，这使得最终的 executable 尽量小
