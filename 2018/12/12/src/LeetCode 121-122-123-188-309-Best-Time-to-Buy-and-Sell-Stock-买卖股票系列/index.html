LeetCode 121|122|123|188|309. Best-Time-to-Buy-and-Sell-Stock-买卖股票系列

vicety · 2018-12-12 · 次阅读

题目

假设你知道股价序列，初始状态不持有股票，每次只能买入一股且在持有股票的同时不能继续买入，可以在同一时间点买入卖出。一次买入+卖出称作一个 transaction，给出股价序列，求最大利润。

Best Time to Buy and Sell Stock I

从头到尾只能包含一个 transaction：找数组最大值减最小值即可，只有一次买入卖出机会，当然选值差距最大的组合。

Best Time to Buy and Sell Stock II

可以使用无限个 transaction：求股价上升量之和即可，也就是成功把握住了每次股价上升的机会。

Best Time to Buy and Sell Stock III

限制只有两次 transaction 机会

由于不能同时 enroll 多个 transaction，这使得 transaction 之间是可分的，令问题 I 的解为solve1(0, arr.length - 1)，那么对于问题 III，solve3 = Max(solve1(0, i) + solve1(i, arr.length - 1) for i in range(0, arr.length))，通过从左往右、从右往左解两遍问题 I，可以 O(n)计算出solve1(0, i) for i in range(0, arr.length)和solve(i, arr.length) for i in arnge(0, arr.length)，solve1(0,i)和solve1(i+1,arr.length)分别可以从从左到右、从右到左两次求解中得到。

class Solution {
    public int maxProfit(int[] prices) {
        int[] left = new int[prices.length];
        int[] right = new int[prices.length];
        int nowAns = 0;
        int min = Integer.MAX_VALUE;
        for (int i = 0; i < prices.length; i++) {
            if (prices[i] < min) min = prices[i];
            else nowAns = Math.max(nowAns, prices[i] - min);
            left[i] = nowAns;
        }
        nowAns = 0;
        int max = Integer.MIN_VALUE;
        for(int i = prices.length - 1; i >= 0; i--) {
            if(prices[i] > max) max = prices[i];
            else nowAns = Math.max(nowAns, max - prices[i]);
            right[i] = nowAns;
        }
        int ans = 0;
        for(int mid = 0; mid < prices.length; mid++) {
            ans = Math.max(ans, left[mid] + right[mid]);
        }
        return ans;
    }
}

Best Time to Buy and Sell Stock IV

最初考虑使用solve4(i, k)代表在时间段[0, i]内完成了 k 次交易时获得的最大收益，类似于 solve3 的解法，分解 k 次 transaction 为k-1 + 1，有solve4(i, k) = max(solve4(j, k - 1) + solve1(j, i) for j in range(0, i + 1))，其中 solve1(i, j)可以用 O(n^2)预处理得到，根据上述递推表达式 solve4(n, k)的复杂度是 O(kn^2)
（防止以后自己看不懂……）
O(n^2)获得solve1(i,j)可以通过dp的方式得到，再套一层k，整体复杂度是O(kn^2)

for i:
    min = INT_MAX
    for j:
        min = max(min, price[j])
        solve1[i][j] = max(solve1[i][j-1], price[j] - min)

这样的复杂度并不能通过面试，但是也没有优化空间了。我认为原因在于递推关系不太满足 DP 中递推关系的原子性，最内层的 max 还需要一个 O(n)循环
于是考虑新的表达方式，solve4(i, k)代表在时间段[0, i]内完成了 k 次交易且最后一次交易在 t=i 时完成（可以发现表示的信息比前面的解法多一些），那么有递推关系solve4(i, k) = max(solve(j, k - 1) + (prices[i] - prices[j])) for j in range(0, i + 1)，尽管这样在外层套了 i 和 k 的循环后仍然是 O(kn^2)的复杂度，但可以发现有进一步的优化空间。可以发现 solve(j, k - 1) - prices[j]被重复使用，于是维护这个表达式（也就是代码中的 prep）的最大值即可，复杂度O(kn)。另外最终要求的是最大利润而非solve4(prices.len - 1, k)的值，因此还要利用dp[k][j] = max(dp[k][j - 1], solve4(j, k)的dp公式。

public int maxProfit(int k, int[] prices) {
    if(prices.length == 0) return 0;
    // if(k > prices.length) k = prices.length / 2;
    if(k > prices.length) {
        int ans = 0;
        for(int i = 1; i < prices.length; ++i){
            if(prices[i] > prices[i - 1])
                ans += prices[i] - prices[i - 1];
        }
        return ans;
    }
    int[][] dp = new int[k + 1][prices.length];
    for (int i = 1; i <= k; i++) {
        int prep = -prices[0];
        // j = 0 -> dp[i][0] = 0
        for (int j = 1; j < prices.length; j++) {
            dp[i][j] = Math.max(dp[i][j - 1], prep + prices[j]);
            prep = Math.max(prep, dp[i - 1][j] - prices[j]);
        }
    }
    return dp[k][prices.length - 1];
}

update at 2020/03

重写了一遍，希望能说得更清楚一些。对于某一时间点t可以选择不交易，那么solve(k, t) = solve(k, t - 1)，同样也可以选择交易，此时对于k次交易可以划分为k-1次与1次，即solve(k - 1, 0, i) - min(price[i, t]) + price[t]，存在两个循环，遍历i是一个，min是另一个。price[t]为固定值，暂时不考虑

假设在j取得min(price[i, t])，那么对于任何solve(k - 1, 0, i) - min(price[i : t])，都一定劣于solve(k - 1, 0, j) - price[j]，由于不清楚这个j在什么位置，采用一个循环求最大值，即max(solve(k - 1, 0, j) - price[j] for j in range(0, t))，此时优化为一个循环

最后，发现max(solve(k - 1, 0, j) - price[j] for j in range(0, t))对t循环时被所有的t复用，因此这一个循环可以优化为O(1)的操作

public int maxProfit(int k, int[] prices) {
     if (prices.length <= 1) return 0;
     if (k >= prices.length) {
         int ans = 0;
         for (int i = 1; i < prices.length; i++) {
             if (prices[i] > prices[i - 1]) ans += prices[i] - prices[i - 1];
         }
         return ans;
     }
     int[][] dp = new int[k + 1][prices.length];
     for (int i = 1; i <= k; i++) {
         int acc = Integer.MIN_VALUE;
         for (int j = 1; j < prices.length; j++) {
             acc = Math.max(acc, dp[i - 1][j - 1] - prices[j - 1]);
             dp[i][j] = Math.max(dp[i][j - 1], acc + prices[j]);
         }
     }
     return dp[k][prices.length - 1];
 }

Best Time to Buy and Sell Stock with Cooldown

无限次交易，限制卖出后需要经过cooldown时间才可再次买入。类似于LC188的思路，对于k时刻买入i时刻卖出操作dp[i] = dp[j] - price[k] + price[i]来说，确保dp[k - cooldown - 1] - price[k]优于其他任何满足冷却时间条件的j where j < k - cooldown - 1，因此dp[i] = price[i] + max([dp[j - cooldown - 1] - price[j] for j in range(j - cooldown)]，其中[dp[j - cooldown - 1] - price[j]被所有dp复用，因此可以缓存。

初始条件：j - cooldown - 1 < 0时dp = 0，其余情况应用递推公式即可

另外由于dp[i]最远依赖dp[i - cooldown - 2]且题目给出的cooldown只有1，可以优化为O(1)空间

public int maxProfit(int[] prices) {
    if (prices.length == 0) return 0;
    int[] dp = new int[prices.length];
    int tmp = 0;
    for (int i = 0; i < prices.length; i++) {
        if (i == 0) tmp = -prices[0];
        else if (i == 1) {
            dp[1] = Math.max(0, prices[i] + tmp);
            tmp = Math.max(tmp, -prices[1]);
        } else {
            // don't sell now or sell with the buying time satisfying the restriction
            dp[i] = Math.max(dp[i - 1], prices[i] + tmp);
            // must sell 3 days before, may buy one day before
            tmp = Math.max(tmp, dp[i - 2] - prices[i]);
        }
    }
    return dp[prices.length - 1];
}
