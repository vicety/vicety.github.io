golang学习-Array、Slice与String

vicety · 2021-3-11 · 次阅读

本文是阅读The Go Blog Arrays, slices (and strings): The mechanics of ‘append’小节后的总结笔记

Arrays

如何初始化数组

var buffer [256]byte
array := [3]int{1, 2, 3}
array = [...]int{1, 2, 3} // 编译器推断数组长度

// 在编译器检查越界访问行为
fmt.Println(buffer[256])
// # command-line-arguments
// ./main.go:<Line of the code>: invalid array index 256 (out of bounds for 256-element array)

array与slice是不同的，array具有固定的长度，这个长度也是array类型的一部分:

var array [256]byte
// array -> type [256]byte
slice := array[255:255]
// slice -> type []byte

Slices

slice可以看做数组指针，slice可以用如下结构体表示

type sliceHeader struct {
    Length        int
    Capacity      int
    ZerothElement *TypeOfArray // slice中第一个元素对应数组中的位置
}

slice的初始化方式

slice := make([]int, 2, 4) // len, cap
slice := make([]int, 2) // len == cap == 2
slice := array[0:0]
slice := []int{1, 2, 3}
slice := []int(nil) // len = cap = 0, appendable

fmt.Println(slice[114514]) // out of bound

// 在运行时panic
// panic: runtime error: index out of range

// goroutine 1 [running]:
// main.main()
    // /Users/gaoce/Github/labenv/golang/main.go:11 +0x111
// exit status 2

slice在函数中是pass by value的，如果需要在函数中修改入参slice的指向，需要传入slice指针，实例略

内置函数

len与cap

len与cap是两个内置函数，接收array或slice作为参数，len返回array或slice或string的当前长度，cap返回array或slice的容量（slice的容量由其底层指向的array与ZerothElement决定，array总有len(arr) == cap(arr)）

copy

看一下下面的插入函数就明白了，此外，copy会自动选择元素少的那个slice进行搬移

// Insert inserts the value into the slice at the specified index,
// which must be in range.
// The slice must have room for the new element.
func Insert(slice []int, index, value int) []int {
    // Grow the slice by one element.
    slice = slice[0 : len(slice)+1]
    // Use copy to move the upper part of the slice out of the way and open a hole.
    copy(slice[index+1:], slice[index:])
    // Store the new value.
    slice[index] = value
    // Return the result.
    return slice
}

append

capacity是不会自动增长的，除非变量指向的array，或者slice内部指向的array发生了变化，append函数会自动做扩容，类似java的ArrayList，由于传入的是slice值而非指针，而内部实现中可能会allocate新空间，因此编译器要求必须接受append函数的返回值（也就是新的slice指向）

String

String仅仅只是read-only的byte slice，此外在语法上提供了一些支持的类型

slash := "/usr/ken"[0]
usr := "/usr/ken"[0:4]
str := string(slice) // slice -> type []byte
slice := []byte(str) // convert back to []byte

string底层的数组是隐藏的，对string来源[]byte的改动不会影响string的值。

早期实现中，对string做的任何修改操作都会导致数组复制，slice的引入使得性能有所改善。
